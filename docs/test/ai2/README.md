“上下文”（Context）是一个广泛应用于多个领域的概念，其核心含义是指围绕某个事物或事件的相关环境、条件、信息或背景。理解上下文对于准确理解和处理信息至关重要。

在不同领域，上下文有其特定的解释：

*   **日常语言中**：上下文通常指“语境”，即语言环境。例如，在理解某个词语或句子的含义时，需要结合它前后的文字来判断其确切意思。如果脱离上下文，同一个词可能会有完全不同的解释。
*   **计算机科学中**：
    *   上下文是指一个任务（如进程或线程）运行所必需的一组数据。这些数据允许任务在被中断后，仍能从中断的地方继续执行。这在处理中断时尤为重要，处理器会保存当前的上下文，以便中断处理完成后恢复任务。上下文数据可能包括处理器寄存器、任务使用的内存以及操作系统管理的控制寄存器等。上下文越小，中断引起的延迟就越小。
    *   在编程中，特别是在一些框架或库中，`Context` 常被用来传递请求范围内的值、取消信号、截止时间等，以管理协程的生命周期或传递请求相关的信息。例如，Go语言的`context`包就定义了这样的类型。
    *   在Spring Cloud等应用中，上下文可以理解为代码运行时的程序环境，它包含了一组状态和外部变量，例如引导上下文（bootstrap context）就是一个保存配置的大对象，作为主应用程序的父上下文来管理配置。
    *   在Java的Servlet中，`ServletContext`定义了Servlet与Servlet容器通信所需的一组方法，它存储了系统的一些初始化信息。
*   **人工智能领域**：
    *   在大型语言模型（LLM）的应用中，“上下文工程”（Context Engineering）是一个新兴且重要的概念。它指的是如何合理地处理和组织提供给模型的信息（即上下文），以最大化模型的性能。这包括了提示词工程（Prompt Engineering），但更广义地涵盖了所有模型能看到的内容。
    *   上下文工程涉及多种手段，如通过RAG（Retrieval-Augmented Generation）技术获取外部信息，利用记忆模块（长时记忆和短期记忆）回顾过往记录，以及通过工具调用（Function Tool/Tool Calling）让模型获取决策所需的信息。

总的来说，上下文是确保某个事物或过程能够正确理解、执行或运行所必需的背景信息、状态或环境。

# JavaScript 数据类型总结

## ✅ 核心结论
- JavaScript 中的数据类型主要分为两大类：原始类型 (Primitive Types) 和对象类型 (Object Type)。
- JavaScript 共有 8 种数据类型，其中 7 种是原始类型，1 种是对象类型。

## 🧠 关键概念
- **原始类型 (Primitive Types)**：
    - **String (字符串)**：用于表示文本数据，由单引号、双引号或反引号包裹的字符序列。
    - **Number (数字)**：用于表示整数或浮点数，不区分整数、浮点数等。包括特殊数值 `NaN` 和 `Infinity`。
    - **BigInt (大整数)**：用于表示和操作超出 Number 类型安全表示范围的非常大的整数。
    - **Boolean (布尔值)**：表示逻辑实体，只有 `true` 和 `false` 两个值。
    - **Undefined (未定义)**：当变量被声明但未赋值时的默认值。
    - **Null (空)**：表示空值或“无”值，是一个只有一个值 (`null`) 的特殊原始类型。
    - **Symbol (符号)**：ES6 中新增的类型，表示独一无二的值。
- **对象类型 (Object Type)**：
    - **Object (对象)**：JavaScript 中所有非原始值的基石，用于存储键值对的集合。数组 (Array) 和函数 (Function) 也是特殊类型的对象。

## 📌 适用场景
- **String**：表示文本信息，如姓名、地址、消息内容等。
- **Number**：表示数值，如年龄、价格、数量、计算结果等。
- **BigInt**：处理需要精确计算的超大整数，例如加密、时间戳等。
- **Boolean**：进行逻辑判断和控制程序流程，如条件语句、循环等。
- **Undefined**：表示变量尚未被赋值的状态。
- **Null**：明确表示一个变量没有值或指向一个空对象。
- **Symbol**：为对象属性创建唯一的键，避免属性名冲突。
- **Object**：存储复杂的数据结构，如用户配置、数据库记录、集合数据（数组）、可执行代码（函数）等。

## ⚠️ 常见误区
- JavaScript 的 Number 类型不区分整数和浮点数，统一使用 Number 类型。
- `typeof null` 会返回 "object"，这是一个历史遗留的 bug。
